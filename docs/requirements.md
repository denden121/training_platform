# Training Platform — Требования к проекту

## 1. Обзор продукта

**Цель**: Платформа для сбора данных о тренировках и восстановлении, с планированием тренировок через ИИ-агента.

**Целевая аудитория (v1)**: Личное использование + друзья
**Будущее**: SaaS-продукт для любителей циклических видов спорта и OCR-гонок
**Платформа**: Web (v1), Mobile (v2)

---

## 2. Виды спорта

- Бег (road, trail)
- Велоспорт (road, MTB, indoor/Zwift)
- Плавание
- Триатлон
- OCR-гонки (Spartan Race, Tough Mudder и аналоги)
- Силовые тренировки (вспомогательные, для OCR)

---

## 3. Пользователи и роли

| Роль        | Описание                                                 |
|-------------|----------------------------------------------------------|
| `athlete`   | Основной пользователь — видит и управляет своими данными |
| `admin`     | Управление пользователями, системные настройки (v2)      |
| `coach`     | Просмотр данных подопечных, ввод корректировок (v3)      |

## 4. Функциональные требования

### 4.1 Аутентификация и профиль

- [ ] Регистрация / вход (email + пароль, OAuth через Google)
- [ ] Профиль спортсмена:
  - Личные данные (возраст, вес, пол)
  - Цели (целевая гонка, дата, дистанция / тип)
  - Доступное время на тренировки (часов/неделя, дни)
  - Фитнес-метрики: FTP, VO2max, пороговый темп бега, порог ПАНО
  - Уровень подготовки (начинающий / любитель / продвинутый)
- [ ] Настройки интеграций с внешними сервисами

### 4.2 Сбор данных о тренировках

#### 4.2.1 Автоматический импорт (интеграции)

| Сервис        | Данные                                      | Приоритет |
|---------------|---------------------------------------------|-----------|
| Garmin Connect | Тренировки, HRV, сон, нагрузка (Training Load) | P1      |
| Strava        | Тренировки, сегменты, мощность              | P1        |
| Apple Health  | Сон, активность, ЧСС                        | P2        |
| Polar Flow    | Тренировки, Orthostatic test, HRV           | P2        |
| Wahoo         | Тренировки, мощность                        | P2        |
| Whoop         | Recovery score, HRV, сон, strain            | P2        |
| Oura Ring     | Сон, Recovery score, HRV                   | P3        |

- Автоматическая синхронизация по расписанию (webhook или polling)
- При конфликте — ручные данные имеют приоритет над автоматическими
- История синхронизации (лог, статус, ошибки)

#### 4.2.2 Ручной ввод и редактирование

- Создание тренировки вручную
- Редактирование любого поля автоматически импортированной тренировки
- Удаление тренировки
- Отметка тренировки как "не засчитанной" (болезнь, форс-мажор)

#### 4.2.3 Структура тренировки

```
Workout
├── Тип (бег, велосипед, плавание, OCR, силовая, другое)
├── Дата и время
├── Длительность (факт)
├── Дистанция
├── ЧСС (средний, максимальный)
├── Мощность (если есть): средняя, нормализованная, IF, TSS
├── Темп / скорость
├── Набор высоты
├── Распределение по зонам (время в каждой зоне ЧСС/мощности)
├── RPE (субъективная нагрузка, 1-10)
├── Заметки / описание
├── Файл (GPX, FIT)
└── Источник (garmin / strava / manual)
```

### 4.3 Сбор данных о восстановлении

- [ ] Ежедневный дневник самочувствия:
  - Качество сна (часы, субъективно 1-5)
  - Усталость мышц (1-5)
  - Общее самочувствие (1-5)
  - Мотивация (1-5)
  - Боли / травмы (свободный текст)
- [ ] Объективные метрики (из интеграций):
  - HRV (утренний)
  - ЧСС покоя
  - Длительность и структура сна (глубокий, REM, пробуждения)
  - Recovery Score (Whoop / Oura / Garmin)
- [ ] Заметки по питанию (свободный текст, опционально)

### 4.4 Планирование тренировок

#### 4.4.1 Плановые тренировки

```
PlannedWorkout
├── Дата
├── Тип
├── Описание / цель тренировки
├── Целевая длительность
├── Целевая нагрузка (TSS / зоны)
├── Структура (разминка / основная часть / заминка с блоками)
├── Статус (planned / completed / skipped / modified)
└── Источник (ai_agent / manual)
```

#### 4.4.2 ИИ-агент планирования

**Принцип работы:**

1. Агент получает через API платформы все данные пользователя:
   - История тренировок (последние N недель)
   - Данные восстановления
   - Текущие метрики (CTL, ATL, TSB, HRV-тренд)
   - Профиль и цели
   - Уже запланированные события (гонки, важные даты)

2. Агент генерирует план (неделя / 2 недели / месяц):
   - Расписание тренировок на период
   - Тип, цель, нагрузка каждой тренировки
   - Общий объём и интенсивность периода
   - Обоснование плана (текст)

3. Пользователь получает предложение и может:
   - Принять план целиком
   - Отредактировать отдельные тренировки
   - Отклонить и попросить другой вариант (с комментарием)

4. После согласования план сохраняется в БД

**Контекст для агента:**
- Принципы периодизации (Base → Build → Peak → Race → Recovery)
- Текущая фаза тренировочного цикла
- Недельный TSS-бюджет исходя из CTL и состояния
- Правила: не более 2 тяжёлых сессий подряд, обязательный восстановительный день после гонки и т.д.

### 4.5 Аналитика и дашборд

- [ ] **Форма (Fitness / Fatigue / Form)**:
  - CTL (Chronic Training Load) — хроническая нагрузка
  - ATL (Acute Training Load) — острая нагрузка
  - TSB (Training Stress Balance) — форма = CTL - ATL
  - График по времени
- [ ] **Объём и нагрузка**:
  - Км / часы по неделям / месяцам
  - TSS по неделям
  - Распределение по типам тренировок
- [ ] **Восстановление**:
  - Тренд HRV
  - ЧСС покоя
  - Средний Recovery Score
- [ ] **Прогресс по метрикам**:
  - FTP история
  - VO2max история
  - Лучшие результаты на дистанциях
- [ ] **Предстоящие события**:
  - Календарь гонок / соревнований
  - Текущий план на неделю

## 5. Нефункциональные требования

### 5.1 Безопасность
- JWT аутентификация (access + refresh токены)
- Данные одного пользователя недоступны другим
- Хранение OAuth токенов к внешним сервисам в зашифрованном виде
- HTTPS only

### 5.2 Производительность
- Синхронизация с интеграциями — фоновые задачи (Celery + Redis)
- API ответ < 300ms для стандартных запросов
- Пагинация для списков тренировок

### 5.3 Надёжность
- Идемпотентность импорта (повторный импорт не дублирует данные)
- Логирование ошибок синхронизации
- Retry-механизм для внешних API

## 7. Интеграции — детали

### Garmin Connect
- OAuth 2.0 авторизация
- Webhooks или Garmin Health API polling
- Получение: Activities, Sleep, HRV Status, Training Status, Body Battery

### Strava
- OAuth 2.0 + webhook events
- Получение: Activities (с потоками мощности и ЧСС)

### Whoop
- OAuth 2.0
- Получение: Recovery, Sleep, Workouts, Cycles
