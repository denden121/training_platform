# Profile — Функциональные требования

## 1. Обзор

Профиль спортсмена — центральная сущность платформы. Данные профиля используются ИИ-агентом при генерации тренировочных планов, расчёте TSS-бюджета, настройке зон интенсивности. Чем полнее профиль — тем точнее план.

---

## 2. Структура данных

### 2.1 Личные данные

| Поле | Тип | Обязательное | Валидация | Примечание |
|---|---|---|---|---|
| `first_name` | string | Нет | max 100 символов | |
| `last_name` | string | Нет | max 100 символов | |
| `birth_date` | date | Нет | — | Используется для расчёта возраста при оценке ЧСС зон |
| `gender` | enum | Нет | male / female / other | |
| `weight_kg` | float | Нет | > 0 | Хранится в кг. UI поддерживает переключение кг ↔ lbs |
| `height_cm` | int | Нет | 50–250 | Хранится в см. UI поддерживает переключение см ↔ ft+in |
| `timezone` | string | Нет | IANA timezone | Дефолт — часовой пояс браузера. Влияет на отображение расписания тренировок |

### 2.2 Фитнес-метрики

| Поле | Тип | Обязательное | Валидация | Описание |
|---|---|---|---|---|
| `training_level` | enum | **Да** | beginner / amateur / advanced / elite | Определяет базовый TSS-бюджет и темп наращивания нагрузки |
| `weekly_hours_available` | float | **Да** | 1–40 | Часов в неделю для тренировок. Лимит объёма плана |
| `ftp_watts` | int | Нет | 50–600 | Functional Threshold Power. Основа для велозон мощности |
| `vo2max` | float | Нет | 20–100 | Максимальное потребление O₂ (мл/кг/мин). Индикатор аэробной мощности |
| `lthr` | int | Нет | 80–220 | Пульс лактатного порога (bpm). Основа для беговых и вело зон ЧСС |
| `hr_max` | int | Нет | 100–230 | Максимальный пульс (bpm) |
| `hr_resting` | int | Нет | 30–100 | Пульс покоя (bpm). Используется для мониторинга восстановления |
| `threshold_pace_sec_per_km` | int | Нет | 120–900 | Пороговый темп бега (сек/км). Основа для беговых зон темпа |

## 3. Уровень подготовки — детализация

| Значение | Критерии |
|---|---|
| `beginner` | До 6 мес. стажа или < 3 тренировок в неделю. Цель — финишировать |
| `amateur` | 1–3 года, 4–8 ч/нед. Участвует в соревнованиях ради удовольствия |
| `advanced` | 3+ лет, 8–14 ч/нед. Целенаправленная периодизация, результат важен |
| `elite` | 14+ ч/нед. Полупрофессионал, топ возрастной группы |

**Влияние на ИИ-агента:**
- Начинающий: правило 10% роста объёма строго, минимум 2 дня отдыха в неделю
- Любитель: стандартная периодизация 3:1 (3 недели нагрузки : 1 восстановление)
- Продвинутый: возможен цикл 4:1, допустимы двухразовые тренировки
- Элита: индивидуальная структура, большой объём, полная периодизация

## 4. Поведение UI

### 4.1 Форма

- Все поля опциональны кроме `training_level` и `weekly_hours_available`
- Обязательные поля отмечены `*` красным
- Кнопка «Сохранить» всегда активна (не блокируется по `isDirty`)
- При успешном сохранении показывается статус «Сохранено» рядом с кнопкой
- Ошибки валидации показываются сразу под полем после попытки сохранения или при изменении значения

### 4.2 Единицы измерения

- **Вес:** переключатель `кг / lbs` рядом с лейблом. Конвертация: 1 кг = 2.20462 lbs. В БД всегда хранится в кг.
- **Рост:** переключатель `см / ft` рядом с лейблом. В режиме ft — два поля (футы + дюймы). Конвертация: 1 дюйм = 2.54 см. В БД всегда хранится в см.
- Выбор единиц — UI-состояние, не сохраняется в БД. При перезагрузке сбрасывается на метрику.

> **TODO:** сохранять предпочтение единиц в `localStorage`.

### 4.3 Часовой пояс

- Компонент с поиском по всем IANA-зонам (`react-timezone-select`)
- Дефолт для нового пользователя — `Intl.DateTimeFormat().resolvedOptions().timeZone`
- Отображение: `(UTC+3:00) Europe/Moscow`
- Названия зон локализуются через `Intl.DateTimeFormat` на текущий язык интерфейса

### 4.4 Дата рождения

- Кастомный календарь (`react-day-picker`) с дропдаунами месяца и года
- Диапазон: 1920 — текущий год
- Формат отображения: `dd.MM.yyyy`
- Локализован через `date-fns/locale`

### 4.5 Подсказки (tooltips)

Поля с `?` иконкой и подсказкой:

| Поле | Содержимое подсказки |
|---|---|
| `training_level` | Описание всех 4 уровней с критериями |
| `weekly_hours_available` | Как используется агентом |
| `ftp_watts` | Что такое FTP, как определить (20-мин тест × 0.95) |
| `vo2max` | Что значит, что Garmin/Polar считают автоматически |
| `lthr` | 30-мин тест, средний пульс последних 20 мин |
| `hr_max` | Почему формула 220−возраст неточна |
| `hr_resting` | Когда измерять, что значит рост |
| `threshold_pace_sec_per_km` | Соответствие темпу на 10 км – полумарафон |

Tooltip рендерится через портал (`@base-ui/react Tooltip`) — не обрезается границами карточки. Задержка показа: 300 мс.

## 5. Локализация

- Поддерживаемые языки: **ru** (дефолт), **en**
- Переключатель языка — кнопка EN/RU в правом верхнем углу
- Язык определяется автоматически из браузера (`i18next-browser-languagedetector`)
- Все строки интерфейса, сообщения об ошибках, единицы измерения, подсказки — в файлах `src/i18n/locales/ru.ts` и `en.ts`
- Ошибки с бэкенда маппятся на локализованные строки (см. `LoginPage.tsx → errorMap`)

---

## 6. API

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/profile` | Получить профиль текущего пользователя |
| `PUT` | `/profile` | Обновить профиль. Принимает частичный объект (все поля опциональны) |

**Правила API:**
- Пустая строка `""` в числовых полях отправляется как `null`
- Единицы всегда в метрике (кг, см) — конвертация только на UI
- При обновлении возвращается полный объект профиля
