# Страница стартов — Требования и функционал

## 1. Назначение

Страница стартов — центральное место управления соревновательным календарём спортсмена.
Старты служат целевыми событиями для ИИ-агента: при генерации плана агент учитывает даты гонок, дистанции и приоритет каждого старта.

---

## 2. Модель данных

```
Race
├── id                        UUID, PK
├── user_id                   UUID, FK → User
├── name                      string(255)         — название гонки
├── sport_type                enum                — тип спорта
├── race_type                 enum                — дистанция / формат
├── date                      date                — дата старта
├── location                  string(255)?        — город / страна
├── distance_km               float?              — дистанция (км)
├── elevation_m               int?                — набор высоты (м)
├── priority                  enum                — A / B / C
├── goal_type                 enum                — finish / time / position
├── goal_time_sec             int?                — целевое время (сек), если goal_type = time
├── goal_position             int?                — целевая позиция, если goal_type = position
├── status                    enum                — upcoming / completed / dns / dnf
├── result_time_sec           int?                — итоговое время (заполняется после гонки)
├── result_position           int?                — итоговая позиция
├── result_total_participants int?                — всего участников (для контекста позиции)
├── notes                     text?               — заметки (подготовка, тактика, итоги)
├── url                       string?             — ссылка на сайт гонки
├── created_at                timestamp
└── updated_at                timestamp
```

### Перечисления

**sport_type:**
- `running` — бег (шоссе / трейл)
- `cycling` — велосипед (шоссе / MTB / indoor)
- `swimming` — плавание
- `triathlon` — триатлон
- `duathlon` — дуатлон
- `ocr` — OCR (Spartan, Tough Mudder и др.)
- `other` — прочее

**race_type** (зависит от sport_type):
- Бег: `5k` / `10k` / `half_marathon` / `marathon` / `ultra` / `trail` / `custom`
- Велосипед: `criterium` / `road_race` / `gran_fondo` / `mtb` / `custom`
- Триатлон: `sprint` / `olympic` / `half_ironman` / `ironman` / `custom`
- OCR: `sprint` / `super` / `beast` / `ultra_beast` / `custom`
- Остальные: `custom`

**priority:**
- `A` — главная гонка сезона (пик формы, максимальная подготовка)
- `B` — важный старт (хорошая форма, без специальной подводки)
- `C` — тренировочная гонка (участие без подводки, как тяжёлая тренировка)

**goal_type:**
- `finish` — просто финишировать
- `time` — уложиться в целевое время
- `position` — занять целевую позицию

**status:**
- `upcoming` — предстоящий
- `completed` — завершён (с результатом)
- `dns` — Did Not Start (не вышел на старт)
- `dnf` — Did Not Finish (не финишировал)

## 3. Страница: список стартов

### 3.1 Отображение

- Список стартов с разделением на **предстоящие** и **прошедшие**
- Карточка старта показывает:
  - Название и тип гонки
  - Дата и локация
  - Дистанция
  - Приоритет (A/B/C — цветовая маркировка: A=красный, B=жёлтый, C=серый)
  - Цель (финиш / время / позиция)
  - Для прошедших — результат и статус (✅ / DNS / DNF)
  - Количество дней до старта (для предстоящих)

### 3.2 Сортировка и фильтрация

- Сортировка по дате (по умолчанию — ближайшие сначала)
- Фильтр по типу спорта
- Фильтр по году
- Фильтр по приоритету (A/B/C)
- Переключатель: предстоящие / прошедшие / все

### 3.3 Виды отображения

- **Список** — карточки с кратким описанием
- **Календарь** — месячный вид с отметками на датах

## 4. Форма добавления / редактирования старта

### 4.1 Поля формы

**Основная информация:**
- Название гонки `*`
- Тип спорта `*` (select)
- Тип дистанции / формат `*` (select, зависит от типа спорта)
- Дата `*` (date picker)
- Локация (город, страна)
- Ссылка на сайт гонки

**Параметры дистанции:**
- Дистанция (км) — заполняется автоматически для стандартных форматов, редактируется
- Набор высоты (м) — опционально

**Планирование:**
- Приоритет `*` (A / B / C) с подсказкой что значит каждый
- Тип цели (финиш / время / позиция)
- Целевое время (появляется при goal_type = time) — ввод в формате ЧЧ:ММ:СС
- Целевая позиция (появляется при goal_type = position)

**Заметки:**
- Свободный текст (подготовка, тактика, ожидания)

### 4.2 После гонки — внесение результата

При статусе `upcoming` → дата прошла → форма предлагает внести результат:
- Статус (completed / dns / dnf)
- Итоговое время (ЧЧ:ММ:СС)
- Итоговая позиция + всего участников
- Заметки (впечатления, что пошло не так / хорошо)

---

## 5. Детальная страница старта

- Вся информация по гонке
- Блок "До старта": X дней / недель
- Блок "Цель": визуализация цели (целевое время с прогресс-баром от текущего PB)
- Блок "Результат" (для прошедших): время, позиция, сравнение с целью
- Блок "Связанный план": ссылка на тренировочный план, созданный под эту гонку
- Кнопка "Создать план под этот старт" → переход в раздел планирования с предзаполненными данными
- История изменений цели (если цель менялась)

## 6. API эндпоинты

```
GET    /races                    — список стартов (с фильтрами)
POST   /races                    — создать старт
GET    /races/{id}               — детали старта
PUT    /races/{id}               — обновить старт
DELETE /races/{id}               — удалить старт
POST   /races/{id}/result        — внести результат после гонки
GET    /races/upcoming           — ближайшие N стартов (для виджета на дашборде)
```

### Query параметры для GET /races:
- `status` — `upcoming` / `completed` / `dns` / `dnf`
- `sport_type` — фильтр по виду спорта
- `priority` — `A` / `B` / `C`
- `year` — год
- `limit`, `offset` — пагинация

## 7. Интеграция с ИИ-агентом

При генерации тренировочного плана агент получает список стартов пользователя:

```json
{
  "upcoming_races": [
    {
      "name": "Spartan Beast Moscow",
      "date": "2026-08-15",
      "sport_type": "ocr",
      "race_type": "beast",
      "distance_km": 20,
      "priority": "A",
      "goal_type": "time",
      "goal_time_sec": 10800,
      "weeks_until_race": 12
    }
  ]
}
```

**Логика агента:**
- Старт приоритета A → строит периодизацию с пиком формы к этой дате
- Старт приоритета B → планирует хорошую форму без специальной подводки
- Старт приоритета C → включает как тяжёлую тренировку, без разгрузочной недели
- Несколько стартов → учитывает интервалы между ними

---

## 8. Виджет на дашборде

Блок "Ближайшие старты" на главной странице:
- Следующие 2-3 гонки
- Название, дата, тип, приоритет
- Обратный отсчёт (X дней)
- Кнопка "Все старты" → переход на страницу стартов

---

## 9. Уведомления (Этап 6)

- За 4 недели до старта A-приоритета: напоминание о разгрузочной неделе
- За 1 неделю до любого старта: напоминание внести актуальные метрики (FTP, вес)
- После даты старта: предложение внести результат

---

## 10. UX-детали

- При создании старта — автозаполнение дистанции для стандартных форматов
  (например, выбрал `marathon` → дистанция = 42.195 км автоматически)
- Цветовая маркировка приоритета во всём интерфейсе (A=красный, B=жёлтый, C=серый)
- Сравнение цель vs результат: зелёный если выполнил, красный если нет
- Возможность добавить старт прямо с дашборда (быстрая форма)
- Пустое состояние с CTA: "Добавь первую гонку, чтобы ИИ мог построить план"
